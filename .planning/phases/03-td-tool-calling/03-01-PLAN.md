# Plan 03-01: Tool Registry And Model Tool Call Slice

## Goal

Prove that an Agent can expose a TD-native tool, receive a provider-style tool call, execute it safely in TD, and append the result back into conversation history.

## Scope

- Add a `ToolRegistry` extension for registering and discovering TD-native tools.
- Emit OpenAI-compatible tool schemas.
- Parse and execute OpenAI-style `tool_calls`.
- Surface invalid tool-call output as recoverable Agent error state.
- Add Agent integration for applying tool result messages to conversation history.
- Add a generated demo tool and MCP bridge action for live UAT.

## Non-Goals

- General-purpose tool authoring UI.
- Multi-step automatic follow-up model turn after tool execution.
- Packaged `.tox` polish.

## Verification

- `python -m pytest -q`
- Generated `demo/demo.toe` opens and stays alive.
- MCP bridge lists tool schemas.
- MCP bridge executes deterministic provider-style tool call.
- Local model emits an actual `tool_calls` response, TD executes it, and Agent history records the tool result.
