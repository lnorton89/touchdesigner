<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Router to local/provider endpoint | Local TD configuration becomes outbound HTTP traffic to a user-selected endpoint. |
| Worker thread to TD main context | Plain worker result payload re-enters TD and may update outputs or invoke a callback. |
| Callback target parameter to TD execution | User-configured callback target can receive structured payloads. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-05 | Information Disclosure | HTTP adapter | high | mitigate | Never include API key values in saved source or README; only resolve source references at runtime and redact sensitive config in status output. |
| T-01-06 | Elevation of Privilege | Callback target invocation | high | mitigate | Invoke only an explicit target and fixed callback method convention; pass structured payload data and do not evaluate arbitrary callback text. |
| T-01-07 | Denial of Service | Local endpoint calls | high | mitigate | Require finite timeout, classify timeout separately, return control immediately after worker submission, and expose running/error state. |
| T-01-08 | Tampering | Worker handoff payload | medium | mitigate | Validate request_id and status before applying results; ignore stale payloads and normalize malformed response shapes. |
| T-01-SC | Tampering | pip installs | high | mitigate | Human checkpoint must verify `httpx` package identity before any install or pinned dependency adoption. |
</threat_model>
