<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TD custom parameters to Router envelope | User-entered endpoint, model, timeout, prompt source, and callback target become executable request configuration. |
| Router source files to distributable TD component | Source-exported scripts may later be embedded in a `.tox` and shared across machines. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-01 | Information Disclosure | Router config constants and README | high | mitigate | Do not store API key values; include only an API key source field name and document that secret storage belongs to Phase 4. |
| T-01-02 | Information Disclosure | Demo defaults and README | medium | mitigate | Use local endpoint defaults and relative TD operator paths only; avoid private filesystem paths per D-16 and D-17. |
| T-01-03 | Tampering | Request envelope construction | medium | mitigate | Validate provider, base URL scheme, timeout range, prompt/message shape, and request id before worker dispatch. |
| T-01-04 | Denial of Service | Retry and running state | medium | mitigate | Track request_id and running state so retry/reset cannot silently overwrite newer state; stale-result handling is completed in Plan 02. |
</threat_model>
