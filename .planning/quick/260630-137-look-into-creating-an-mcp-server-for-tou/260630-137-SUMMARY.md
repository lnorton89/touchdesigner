### mcp_td_sketch.py
- MCPServer v2 API server with streamable-http transport on `127.0.0.1:8765`
- Three `@mcp.tool()` decorated functions: `td_get_parameter`, `td_get_dat_text`, `td_pulse_trigger`
- Each tool documents real TD equivalent and mock return shapes
- Module-level docstring with curl-based verification command
- `__main__` guard, import-safe (graceful fallback when `mcp` not installed)
- Python 3.11 syntax, compiles cleanly via `py_compile`

## Key Decisions

1. **Companion process > embedded thread:** ASGI event loop inside TD is too risky for v1. Separate process isolates crashes and avoids frame interference.
2. **streamable-http > SSE:** Simpler protocol, sufficient for localhost MCP use. SSE adds complexity without benefit for single-client local scenarios.
3. **stdio rejected:** TD owns process stdin/stdout — stdio-based MCP is structurally incompatible.
4. **Dependency bumps needed:** pydantic `>=2.11.0` and pydantic-settings `>=2.5.2` when `mcp` is added to requirements.txt.
