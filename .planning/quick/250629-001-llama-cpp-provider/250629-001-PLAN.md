# Quick Task 250629-001: Add llama.cpp provider support

**Goal:** The touchdesigner project can use the llama.cpp server managed by the companion
llama project (`documents/dev/llama`) as a first-class provider option.

## Tasks

1. **Add llama.cpp as valid provider in router_http.py**
   - Add `LLAMACPP_BASE_URL = "http://127.0.0.1:8080/v1"` constant
   - Add `"llama.cpp"` to the valid provider set in `build_request_envelope`
   - Use `LLAMACPP_BASE_URL` as default when provider is `"llama.cpp"`
   - Files: `td_components/llm_model_router/router_http.py`

2. **Create convenience launcher script**
   - Create `scripts/start-llama-server.ps1` that delegates to the llama project's
     `scripts/server/start-server.ps1`
   - Resolve llama project path relative to script location with fallback to known path
   - Support -Profile, -Model, -Port, -DryRun parameters
   - File: `scripts/start-llama-server.ps1`

3. **Update documentation**
   - Update `td_components/llm_model_router/README.md` with Provider options
     including `llama.cpp` and its default endpoint
   - Add `scripts/` directory as needed

4. **Add tests**
   - Verify `llama.cpp` is accepted as a valid provider
   - Verify default base URL resolves to `http://127.0.0.1:8080/v1`
   - Verify invalid providers are rejected
   - Verify existing tests still pass
