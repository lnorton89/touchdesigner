The bridge class manages a daemon-thread HTTP server:

**Server startup:**
```python
def start(self) -> None:
    """Start the bridge HTTP server in a daemon thread."""
    # If already running, skip
    server = http.server.HTTPServer((config.BRIDGE_HOST, config.BRIDGE_PORT), _BridgeRequestHandler)
    server.timeout = 1.0  # allow periodic shutdown checks
    # Store reference to self on the server for handler access
    server.bridge = self
    self._server = server
    thread = threading.Thread(target=server.serve_forever, name="MCPBridge-HTTP", daemon=True)
    thread.start()
```

**Request handler** (`_BridgeRequestHandler`, a nested or module-level class extending `http.server.BaseHTTPRequestHandler`):
- Parses URL path: `/td/{route_name}?param1=v1&param2=v2`
- Lookup route name in `MCPBridge_config.ROUTES` to get handler method name
- Call `getattr(server.bridge, handler_method)(parsed_params)`
- Return JSON response with appropriate status code

Handler methods on MCPBridge — each uses the `td.run()` pattern from ModelRouterExt for safe TD API access from the daemon thread:

```python
def _run_on_main(self, func, *args, timeout=5.0):
    """Execute a function on TD's main thread via td.run() and wait for result."""
    result_holder = {}
    error_holder = {}
    event = threading.Event()
    def wrapper():
        try:
            result_holder['data'] = func(*args)
        except Exception as exc:
            error_holder['error'] = str(exc)
        finally:
            event.set()
    try:
        run_func = globals().get("run")  # TD built-in
        if callable(run_func):
            run_func("args[0]()", wrapper, endFrame=True, delayFrames=0)
        else:
            wrapper()  # fallback: execute directly (testing outside TD)
    except Exception as exc:
        error_holder['error'] = str(exc)
        event.set()
    if not event.wait(timeout=timeout):
        raise TimeoutError(f"td.run() did not complete within {timeout}s")
    if 'error' in error_holder:
        raise RuntimeError(error_holder['error'])
    return result_holder.get('data')
```

**6 handler methods** — each validates params, calls td.op() API, returns dict result:

1. `get_parameter(params)` → `op(params['operator_path']).par[params['parameter_name']].eval()` → `{"operator": ..., "parameter": ..., "value": ..., "type": ...}`
2. `get_dat_text(params)` → `op(params['operator_path']).text` → `{"operator": ..., "text": ...}`
3. `pulse_trigger(params)` → `op(params['operator_path']).par[params['pulse_name']].pulse()` → `{"operator": ..., "pulse": ..., "triggered": True}`
4. `list_network_children(params)` → iterate `op(params['operator_path']).children`, filter by `type_filter` if provided, return `{"operator": ..., "children": [{"name": ..., "type": ..., "path": ...}, ...], "count": ...}`
5. `set_parameter(params)` → coerce `value` string (try float, then str), assign `op(params['operator_path']).par[params['parameter_name']] = value` → `{"operator": ..., "parameter": ..., "previous_value": ..., "new_value": ...}`
6. `read_chop_channel(params)` → `op(params['operator_path']).chan(params['channel_name'])`, slice by start/count → `{"operator": ..., "channel": ..., "samples": [...], "length": ...}`

**Conventions:** Use `td.op()` NOT `op()` (TD's global is `td`). Handle missing operators/parameters with clear error messages. The bridge is stdlib-only — no external dependencies.

**HTTP response format:** Always return `{"status": "ok", ...}` on success, `{"status": "error", "error": "..."}` on failure, with HTTP 200 for ok and 400/404/500 for errors.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client → MCP server | Untrusted MCP clients connect to streamable-http on localhost:8765 |
| MCP server → TD bridge | Companion process calls bridge HTTP API on localhost:9876 |
| TD bridge → TD API | Bridge handler dispatches to td.op() on TD main thread |
| External venv → host | mcp package and transitive deps (pywin32, uvicorn, starlette) run in companion process |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-quick-01 | Tampering | td_set_parameter tool | medium | mitigate | Bridge coerces values (try float, then str) and validates operator_path format before calling td.op(). No arbitrary Python eval. |
| T-quick-02 | Information Disclosure | td_get_dat_text, td_get_parameter tools | low | mitigate | Bridge binds to 127.0.0.1 only — no external network exposure. All TD data stays on loopback. |
| T-quick-03 | Spoofing | MCP server → TD bridge | low | accept | Both processes on localhost; no auth between them. Acceptable for local development; future enhancement for production auth. |
| T-quick-04 | Denial of Service | TD bridge HTTP server | low | accept | Single-threaded http.server with 5s td.run() timeout prevents runaway ops. Bridge runs in daemon thread — crash doesn't kill TD. |
| T-quick-05 | Elevation of Privilege | td_execute_script (not implemented) | high | mitigate | Deliberately excluded the `td_execute_script` tool from this implementation — it would allow arbitrary code execution in TD's process. Not exposed. |
| T-quick-SC | Tampering | pip install mcp>=1.27 | high | mitigate | mcp package from PyPI published by Anthropic, PBC (verified in RESEARCH.md Package Legitimacy Audit). MIT license. High source reputation. All transitive deps (starlette, uvicorn, pywin32, etc.) are well-known packages. slopcheck gating not needed for this set. |
